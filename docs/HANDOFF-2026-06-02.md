## What was learned (the most valuable lines)

1. **The reference/portal vs portal-app fork trap.** Another agent (branch
   `claude/focused-gates-0NNMW`, commit `108290583`) improved the **`reference/portal`
   snapshot** in parallel. The live UI is **`portal-app/`** (forked from it in #23). So
   their edits had *zero effect* on the live app — BUT their Tailwind channel-var change
   caught a **real bug in `portal-app`**: accents were `var(--color-accent-*)`, so
   `bg-aurum/10` compiled to `rgb(<full-color> / 0.1)` = invalid CSS → ~15 components'
   accent tints silently dropped. **Lesson: when another agent touches `reference/`,
   check whether it applies to the live `portal-app/` and port it there.** (#31.)
2. **Bug-hunt sweeps (4 total) — mostly false positives; verify before fixing.** Real
   finds: the MCP `err.message` leak (#30) + the profile-editing gap (#30). **False
   positives I disproved:** "ON CONFLICT(territory_id,user_id) vs UNIQUE(user_id,
   territory_id) throws" → SQLite matches the conflict target by **column set**, order-
   independent (proven by passing `verify:chronicles`); "jobs single-flight race" →
   `startClusteringJob` is fully **synchronous** through check→set (no `await`); "stuck
   enrichment state-2 rows" → **self-heals** (state 2 *is* the NLP queue); "scope-denial
   ciphertext leak" → can't trigger in single-user V1 (one scope). **Always reproduce a
   finding empirically before changing code.**
3. **Encrypted columns can't be SQL-filtered.** `description_version` is in
   ENCRYPTED_FIELDS, so `WHERE description_version != ?` always "differs" (ciphertext is
   non-deterministic). Version-gating must compare the **decrypted** value in JS
   (`pipeline/describe-chronicles.js` `getTerritoriesToNarrate`). Also found+fixed
   `getNeedingDescription` selecting a non-existent `open_questions` column. (#28.)
4. **Tier-2 boundary holds.** Generate (clustering) + chronicle narration have their
   **job lifecycle / logic Tier-1 verified** here against fakes/stubs; the **real** runs
   need the host ML stack (FAISS/Leiden/ONNX) + a model (Ollama/BYOK) on the Mac.
5. **`user_profiles` is plaintext** (not in ENCRYPTED_FIELDS) — handle/display_name/
   signature are public-facing, so profile editing writes it directly. (#30.)
6. **Tier-1 verify ≠ the real journey works.** The Mac-test session (#32→#34) found
   two bugs my verify passed over: the **frontend upload path** (`chunked-upload.ts`
   posted raw `/portal/upload`, bypassing the api.ts rewrite → 404 — my verify hit the
   *server* endpoint directly) and the **spawned clustering child's dbPath** (defaulted
   to an empty DB). **Lesson: verify the seam the *frontend/child actually uses*, not
   just the endpoint in isolation.** Integrating another agent's PR also broke a test
   stub (`verify-enrich` needed `embedBatch`) — re-run the WHOLE suite after a merge.

## Gotchas (2026-06-02)

- **Branch off `origin/main`** — local `main` is the legacy `@mycelium/agent-framework`
  lineage (`git checkout main` lands there).
- **`reference/portal` edits ≠ live UI.** The live app is `portal-app/` (forked in #23).
  Port reference-snapshot UI changes into `portal-app/`.
- **Encrypted columns** (per `src/crypto/crypto-local.js` ENCRYPTED_FIELDS) **can't be
  SQL-filtered/ordered** — ciphertext is non-deterministic. Filter the decrypted value in JS.
- **SQLite ON CONFLICT matches by column set**, order-independent — `ON CONFLICT(a,b)`
  works against `UNIQUE(b,a)`. (Not a bug; a sweep flagged it as one.)
- **Remote branch delete = 403** in this environment; user prunes on GitHub.
- **Portal `build/` is gitignored** — `npm run portal:build` after pulling to get the
  canonical UI + the accent fix; else the server serves the legacy single-file portal.
- **No browser/Mac/Tier-2 here** — UI visual correctness, real MCP client, and real
  clustering/embeddings/model runs are the operator's Mac surface.
